# 不动产数据 Skill — CSV→Notebook 转换规范

## CSV→ipynb 转换流程

1. 读取 CSV (utf-8-sig 编码)
2. 分页 (50-80 行/页)
3. 每行每列生成 HTML `<td>`，评分列注入色阶背景色
4. 封装为 Jupyter Notebook JSON (`nbformat: 4`)
5. 提交到 GitHub，自动渲染

## 色阶规则

### 评分列 (0-10 分): 红→黄→绿
```python
ratio = max(0, min(1, (score - 3) / 7))
if ratio < 0.5:
    r, g, b = 220, int(60 + ratio*2*180), 60     # 红→黄
else:
    r, g, b = int(220 - (ratio-0.5)*2*180), 200, int(60 + (ratio-0.5)*2*40)  # 黄→绿
style = f"background-color: rgba({r},{g},{b}, 0.35)"
```

适用列: 居住单身体验, 居住家庭体验, 交通可达性(地铁), 开车舒适度,
性价比, 房屋质量口碑, 附近大厂科技园区, 租房流动性指数,
买卖流动性指数, 5km商业综合指数, 生活便利度, 买菜便利度,
医疗水平, 教育资源指数, 综合评分

### 口碑方差列 (0-5): 绿→红 (反向!)
```
稳定(0-1.5) = 绿色背景 → 评价一致，品质可信
中等(1.5-2.5) = 黄色
分化(2.5-5) = 红色背景 → 争议大，风险高
```

### 价格列: 不着色，右对齐
```html
<td style="text-align:right">¥{int(value):,}</td>
```
适用: 所有 `挂牌均价`, `成交均价`, `租金` 列

### 小区名: 加粗
```html
<td><b>{name}</b></td>
```

## 格式规范

| 元素 | 完整数据表 | 排行榜 |
|------|-----------|--------|
| 字体大小 | 8-9px | 11px |
| 每页行数 | 50-80 | 15-30 |

## 排行榜 Notebook 结构

每个排行表必须包含:
- 小区名 (加粗)
- 区
- 2026 挂牌均价 (¥格式)
- **性价比** (色阶)
- **综合评分** (色阶)

根据主题额外加:
- 家庭: 教育资源指数, 医疗水平
- 单身/科技: 大厂科技园区, 地铁, 2026 租金
- 新盘: 房屋年代, 质量口碑
- 各区统计: 小区数, 均价, 租金, 综合均分 + 条形图

## 48 列 CSV 规范

### 基础信息 (7列)
序号, 区, 板块, 小区名, 房屋年代, 经度, 纬度

### 价格 (21列: 2020-2026 × 3)
{year}挂牌均价, {year}成交均价, {year}租金
- 新盘交付前年份留空
- 成交均价 ≈ 挂牌价 × 0.92

### 评分 (15列, 0-10分)
居住单身体验(5%), 居住家庭体验(8%), 交通可达性(地铁)(10%),
开车舒适度(5%), 性价比(10%), 房屋质量口碑(10%),
附近大厂科技园区(5%), 租房流动性指数(5%), 买卖流动性指数(5%),
5km商业综合指数(8%), 生活便利度(8%), 买菜便利度(5%),
医疗水平(8%), 教育资源指数(—), 口碑方差(反向8%)

### 其他 (5列)
5年趋势(涨/跌/维持), 商业距离(km), 综合评分, 备注, 数据来源

## 数据质量要求

1. **小区名必须真实** — 在链家/贝壳/安居客/房天下可查到
2. **坐标在上海范围内** — lat: 30.6-31.55, lng: 120.8-122.1
3. **区板块对应正确** — 临港→浦东新区, 石化→金山, 四团→奉贤
4. **价格合理** — 2026 成交价不应比 2025 骤降 50%+ (除非有数据来源)
5. **评分 0-10** — 不超范围
6. **高德 API 配额** — 搜索 5000/月, LBS 150000/月, 并发 3/秒
7. **Key 严禁写入文件** — 只通过 `os.environ` 读取
